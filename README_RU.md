# Патчи ACPI DSDT для Lecoo Pro 14 (Linux)

Дамп DSDT и набор патчей для **Lecoo pro 14 (N155A)**.
Патч исправляет следующие ошибки:
- Самопроизвольное включение после выключения на Linux ([Shutdown turns into Reboot](https://4pda.to/forum/index.php?showtopic=1108499&st=3420#entry141777966)).
- Также разблокирует **S3 state** (suspend to RAM), который [аппаратно скорее всего не работает](https://4pda.to/forum/index.php?showtopic=1108499&st=3340#entry141732975)
- Исправляет **переполнение буфера** (buffer overflow) в методах WMI.
- Устраняет ошибки вида "ACPI ...device not found" при загрузке системы.
- **Чинит мгновенное пробуждение** (instant wake) из состояния гибернации (S4).


### ⚠️ Важно
Файл `acpi_override.cpio` должен загружаться **ПЕРЕД** основным образом initramfs.

#### ℹ️ Известные ограничения и проблемы

- **Secure Boot:** Этот патч, скорее всего, не будет работать «из коробки», если включена функция Secure Boot. В зависимости от используемого загрузчика загрузка пользовательских таблиц ACPI при включенной функции Secure Boot может потребовать сложной ручной процедуры подписи и регистрации собственных ключей или изменения файлов в критически важных разделах. Настоятельно рекомендуется отключить функцию Secure Boot перед использованием этого патча, чтобы избежать сбоев в работе системы из-за неправильной настройки. Вам необходимо самостоятельно устранять проблемы или ошибки, возникающие в процессе настройки.

- **TPM 2.0:** Этот патч может привести к тому, что встроенные драйверы ядра Linux не смогут обнаружить fTPM (устройства /dev/tpm0 или /dev/tpmrm0 будут выдавать сообщение о таймауте в dmesg). Это приведет к увеличению времени загрузки на 90 секунд. На данный момент единственным решением является отключение TPM в BIOS. Если вы уже используете функции, связанные с TPM, такие как автоматическая разблокировка разделов LUKS, не рекомендуется использовать этот патч. Однако приветствуются PR, если вы нашли другое решение. (См. [проблему № 3](https://github.com/LaVashikk/lecoo-linux-acpi-fix/issues/3) для справки)


## Подготовка

1. Скачайте файл **`acpi_override.cpio`** из раздела [Releases](https://github.com/LaVashikk/lecoo-linux-acpi-fix/releases).
2. Скопируйте его в корень вашего загрузочного раздела (обычно `/boot`).

```bash
sudo cp acpi_override.cpio /boot/
```
Далее выберите инструкцию для вашего загрузчика.

## Способ 1: Systemd-boot

1. Найдите конфигурационный файл вашего загрузчика. Обычно он находится в `/boot/loader/entries/` и имеет расширение `.conf` (например, `arch.conf` или `linux.conf`).
2. Откройте его для редактирования:
```bash
sudo nano /boot/loader/entries/arch.conf
```

3. Добавьте строку `initrd /acpi_override.cpio` **ПЕРЕД** строкой основного initramfs.
**Пример того, как должно получиться:**
```text
title   Arch Linux
linux   /vmlinuz-linux
initrd  /acpi_override.cpio       # <-- СНАЧАЛА override!!
initrd  /initramfs-linux.img
options root=UUID=... rw

```
4. Сохраните файл (`Ctrl+O`, `Enter`) и закройте (`Ctrl+X`).
5. Готово. Перезагружайтесь.

## Способ 2: GRUB

Для GRUB лучше использовать специальную переменную, чтобы настройки не слетели при обновлении ядра.

1. Откройте файл настройки GRUB:
```bash
sudo nano /etc/default/grub
```

2. Найдите переменную `GRUB_EARLY_INITRD_LINUX_CUSTOM`.
* **Если она есть:** Добавьте туда наш файл.
* **Если её нет:** Добавьте строку в конец файла:

```bash
GRUB_EARLY_INITRD_LINUX_CUSTOM="acpi_override.cpio"
```

*(Примечание: Путь указывается относительно корня /boot. Если не сработает, попробуйте `"/boot/acpi_override.cpio"`).*
3. Обновите конфигурацию GRUB:
* **Arch Linux:**
```bash
sudo grub-mkconfig -o /boot/grub/grub.cfg

```
* **Ubuntu/Debian:**
```bash
sudo update-grub

```

## Способ 3: Limine
Убедитесь, что в вашей системе установлены пакеты `limine-dracut-support` или `limine-mkinitcpio-hook`.
1. Найдите и откройте файл конфигурации Limine. Обычно он находится в `/boot/limine.conf`.
2. Найдите основную запись загрузки (обычно это первая запись) и добавьте новую строку `module_path` для патча перед initramfs. В качестве примера мы воспользуемся CachyOS:
```conf
/+CachyOS
  //linux-cachyos
  ### This kernel entry is auto-generated by limine-entry-tool
  comment: Kernel version: X.X.X.X-X-cachyos
  comment: kernel-id=linux-cachyos
  protocol: linux
  module_path: boot():/acpi_override.cpio   # <-- Введите здесь путь к новому модулю; «boot():/» означает корневой каталог раздела загрузки.
  module_path: boot():/(random path)/linux-cachyos/initramfs-linux-cachyos#{HASH}
  path: boot():/(random path)/linux-cachyos/vmlinuz-linux-cachyos#{HASH}
  cmdline: quiet nowatchdog splash rw rootflags=subvol=/@ root=UUID=XXX
```
3. Перезагрузите систему и проверьте, все ли работает правильно. 
4. Если всё работает правильно, вы можете изменить значение KERNEL_CMDLINE в файле `/etc/default/limine`, чтобы патч оставался в силе после обновления ядра.
```text
# Добавьте путь к патчу в качестве initrd в начало строки; при обновлении загрузчика хуки автоматически преобразуют его в module_path.
# «[default]» означает, что параметры будут применены ко всем установленным ядрам. 
# Если вы хотите применить его только к определенному ядру, вы можете заменить его на имя соответствующей записи в списке загрузки, например «linux-cachyos».
                          ↓  
KERNEL_CMDLINE[default]+="initrd=/acpi_override.cpio quiet nowatchdog splash rw rootflags=subvol=/@ root=UUID=XXX"
```
5. Запустите команду `sudo limine-update`, чтобы применить изменения.

    <summary>Настройка патча с функцией Secure Boot с помощью Limine (для опытных пользователей)</summary>

Чтобы использовать этот патч с функцией Secure Boot, в прошивку необходимо записать пользовательский защитный ключ. Обычно эти действия нужно выполнить только один раз:

> Данный раздел основан на информации из [Руководства по настройке безопасной загрузки CachyOS](https://wiki.cachyos.org/configuration/secure_boot_setup/#enabling-setup-mode-in-uefi) и [ArchWiki — Limine](https://wiki.archlinux.org/title/Limine). Рекомендуем ознакомиться с ними перед началом настройки.

<details>
  <summary>Ознакомьтесь с инструкциями</summary>

1. Загрузитесь в настройки BIOS и перейдите в раздел `Security > Secure Boot > Expert Key Management > Factory Key Provision`. Установите значение `Disable`, а затем нажмите F4, чтобы сохранить настройки и перезагрузить компьютер.
2. Вернитесь в настройки BIOS, перейдите в раздел `Security > Secure Boot > Reset to Setup Mode`, а затем выберите «Yes». Модуль безопасной загрузки BIOS должен сейчас находиться в режиме Setup.
3. Загрузите систему и проверьте, установлен ли sbctl. Если нет, установите его с помощью следующих команд:
```
sudo pacman -S sbctl
```
4. Выполните команду `sudo sbctl status`, чтобы проверить, действительно ли Secure Boot перешел в режим «Setup».
5. Создайте ключи, выполнив команду `sudo sbctl create-keys`
6. Зарегистрируйте новые ключи в EFI, выполнив команду `sudo sbctl enroll-keys --microsoft --firmware-builtin`.
7. Снова выполните команду `sudo sbctl status`, чтобы убедиться, что ключи были успешно установлены.
8. Чтобы включить пользовательскую регистрацию контрольных сумм, установите для следующих параметров значение «yes» в файле `/etc/default/limine`:
```
ENABLE_ENROLL_LIMINE_CONFIG=yes
```
9. Воспользуйтесь приведенными выше инструкциями, чтобы добавить `KERNEL_CMDLINE` для данного патча.
10. Сгенерируйте контрольные суммы BLAKE2B для дополнительных обоев и модулей, поскольку по умолчанию они не подписаны, а затем добавьте их в файл конфигурации Limine, не забыв поставить символ `#` перед контрольной суммой:
```
sudo b2sum /boot/limine-splash.png | cut -d' ' -f1
```
```
sudo b2sum /boot/acpi_override.cpio | cut -d' ' -f1
```
```
# /boot/limine.conf
wallpaper: boot():/limine-splash.png#{YOUR HASH}

module_path: boot():/acpi_override.cpio#{YOUR HASH}
```
**ВНИМАНИЕ: Неверная контрольная сумма приведет к сбою загрузчика!!** Limine необходимо подписать только свои бинарные файлы EFI, чтобы система загружалась с поддержкой Secure Boot; остальные файлы проверяются внутри загрузчика. Поэтому вам нужно лишь убедиться, что контрольные суммы BLAKE2B для всех файлов модулей в конфигурации Limine верны.

11. Сохраните файл и выполните команды `sudo limine-enroll-config` и `sudo limine-update`, чтобы записать конфигурацию в загрузчик.
12. Откройте файл `/boot/limine.conf`, чтобы проверить, есть ли у каждого модуля контрольная сумма BLAKE2B.
13. Перезагрузите компьютер, войдите в BIOS, включите функцию Secure Boot, а затем перезагрузитесь ещё раз, чтобы убедиться, что всё работает правильно. После установки обновления вам необходимо самостоятельно добавить новые контрольные суммы в конфигурационный файл, следуя инструкциям из пунктов 10 и 11.

## Как проверить, что патч работает

После перезагрузки откройте терминал и введите:

```bash
sudo dmesg | grep ACPI | grep override
```

Вы должны увидеть строкии типа:
```
ACPI: Table Upgrade: override [DSDT-ALASKA-  A M I ]
ACPI: DSDT 0x00000000969E3000 Physical table override, new table: 0x000000009B5A5000
```
Если строка есть — патч применился. Попробуйте выключить ноутбук.
